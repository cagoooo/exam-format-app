## 二、目前 P0 清單

P0 定義：直接影響「線上系統能不能被老師放心拿來轉正式考卷」的項目。

| 編號 | 項目 | 目前狀態 | 為什麼重要 | 建議開發內容 | 驗收標準 |
|---|---|---:|---|---|---|
| P0-01 | 圖片保真與安全縮放 | 待做 | 考卷常有圖片、閱讀圖、自然圖表，現在抽文字表格為主，圖片可能流失 | 讀取 DOCX media，重新插入圖片；超過欄寬時等比例縮放；避免圖片跨欄爆版 | 含圖片考卷轉換後圖片不遺失、不超出欄寬 |
| P0-02 | Word 公式與特殊符號保留 | 待做 | 數學與自然考卷常有分數、上下標、OMML 公式 | 保留 OMML，無法保留時轉為高解析圖片 fallback | 公式不消失，PDF 預覽可正確顯示 |
| P0-03 | 國語注音 ruby 保留 | 待做 | 國語考卷、國字檢測常需要注音 | 解析 `<w:ruby>`，輸出時保留注音或轉成穩定文字排版 | 注音題轉換後仍可閱讀，位置不亂 |
| P0-04 | 表格版面保真 | 部分完成 | 很多考卷使用表格排版，跑版常發生在表格 | 保留合併儲存格、欄寬、框線、對齊；自動壓縮超寬表格 | 複雜表格轉換後不破格、不超頁 |
| P0-05 | 超頁原因診斷 | 待做 | 只知道超過 2 頁還不夠，老師需要知道哪裡造成超頁 | 回報最可能造成超頁的段落、表格、圖片、長題組 | 系統能指出「第 N 個表格過寬」或「第 N 段過長」 |
| P0-06 | 兩頁鎖定演算法升級 | 第一版完成 | 現在只有基礎壓縮，正式考卷需要更細膩 | 依序微調字級、行距、段距、表格內距、圖片尺寸，並記錄調整差異 | 多數 2 至 3 頁邊界案例可壓回 2 頁 |
| P0-07 | 線上檔案安全與自動清理 | 待做 | 考卷屬敏感資料，線上暫存需要明確清理策略 | 每次轉換使用獨立 job id；檔案 30 分鐘自動刪除；下載 URL 過期 | 舊檔不可長期存取，暫存區可自動清乾淨 |
| P0-08 | 錯誤訊息中文化與復原建議 | 待做 | 老師遇到失敗時，需要看得懂的原因 | LibreOffice 轉檔失敗、檔案太大、格式不支援、PDF 產生失敗時給中文建議 | 失敗訊息可直接告訴老師下一步怎麼做 |
| P0-09 | 線上轉換壓力與逾時控制 | 待做 | Cloud Run 有冷啟動與逾時限制，正式使用要穩 | 增加檔案大小限制提示、進度狀態、逾時處理、佇列策略 | 大檔不會無聲失敗，前端有明確等待與錯誤狀態 |
| P0-10 | 線上端到端測試 | 部分完成 | 部署後需防止下次改版壞掉 | 建立 GitHub Actions 或腳本測 Cloud Run `/api/analyze`、`/api/convert` | 每次 push 後可自動確認線上轉換可用 |

## 四、未來優化改良建議

### A. 版面穩定性

1. 目標頁數可選 1、2、3 頁或不限制。
2. 自動壓縮策略分級：輕度、中度、強力。
3. 自動偵測孤行、段落標題落在頁尾、大題被切開。
4. 表格超寬時先縮表格內距，再縮字，再調欄寬。
5. 圖片超寬時等比例縮放，必要時移到下一欄或下一頁。
6. 每次壓縮都回報調整內容，例如「行距 15pt -> 14pt」。

### B. Word 內容保真

1. 保留圖片、框線、合併儲存格、底色、對齊方式。
2. 保留粗體、底線、斜體、上下標、特殊符號。
3. 保留 OMML 公式與 ruby 注音。
4. 保留頁首頁尾、頁碼、密封線。
5. 保留題號層級與項目符號。
6. 對無法保留的元素提供警告清單。

### C. 線上系統可靠度

1. 每次轉換建立 job id，避免檔名衝突。
2. 轉換完成後 30 分鐘自動刪除暫存檔。
3. 建立 `/health` 與 `/api/status/<job_id>`。
4. 大檔案顯示進度與等待狀態。
5. Cloud Run 最小實例可視使用量決定是否開啟，降低冷啟動。
6. 記錄錯誤類型，優先修最常見問題。

### D. 教師體驗

1. 首頁直接提供「上傳 -> 選模板 -> 預覽 -> 下載」四步流程。
2. 預設模板記住上次選擇。
3. 提供「國語 / 數學 / 自然 / 社會 / 國字檢測」快速入口。
4. 下載檔名自動加入科目、年級、日期。
5. 顯示「這份考卷目前適合列印」或「可能超頁」的明確狀態。
6. 提供一鍵下載 DOCX + PDF 的 zip。

### E. 安全與隱私

1. 首頁明確說明檔案只用於轉換，不永久保存。
2. 加入檔案自動刪除策略。
3. 若要全校使用，加入 Google 登入。
4. 限制上傳大小與副檔名。
5. 防止公開下載連結被長期存取。

## 五、建議下一步開發順序

### 路線 1：先讓「真實考卷」成功率大幅提高

1. P0-01 圖片保真與安全縮放
2. P0-04 表格版面保真
3. P0-02 Word 公式與特殊符號保留
4. P0-03 國語注音 ruby 保留

適合目標：讓更多老師直接丟自己的考卷進來也能成功轉。

### 路線 2：先讓線上服務更放心

1. P0-07 線上檔案安全與自動清理
2. P0-08 錯誤訊息中文化與復原建議
3. P0-09 線上轉換壓力與逾時控制
4. P0-10 線上端到端測試

適合目標：讓系統從「可以用」變成「敢給同事用」。

### 路線 3：先做成校內正式產品

1. P1-01 老師自訂模板管理
2. P1-02 石門國小模板庫
3. P1-04 Google 登入
4. P1-05 私有檔案儲存
5. P1-14 管理後台

適合目標：從阿凱老師個人工具變成校內共用服務。
